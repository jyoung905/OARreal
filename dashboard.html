Local setup

Use the password from .env. This is a lightweight local gate for internal intake review, not full production authentication.

1. Configure it

Copy .env.example to .env, set a dashboard password, then restart node server.js.

2. Verify it

Log in here, or post to /api/dashboard/login with your password to verify the flow before using the inbox.

3. Keep it local

Keep this dashboard local by default. If you ever allow remote access, use a real password and understand the exposure first.

Quick local check curl -i -X POST http://localhost:8080/api/dashboard/login -H 'Content-Type: application/json' -d '{"password":"your-password"}'
