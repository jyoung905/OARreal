### 2. Mobile navigation fixed
**Files:** `components/SiteHeader.jsx`, `app/globals.css`

The entire `nav-actions` div — including the "Start Your Review" CTA button — was tagged `desktop-only`, meaning nothing appeared in the mobile header. Conversion-critical for mobile traffic.

Fixed by:
- Adding a second CTA button with class `mobile-only` that shows only on screens ≤ 960px
- Keeping the `desktop-only` div (which contains all nav links + desktop CTA) as-is
- Updating the responsive CSS so the nav stays as a row on mobile instead of stacking vertically
- Adding `.mobile-only { display: none }` + a breakpoint override to show it

---

### 3. Email + Telegram notifications wired up
**Files:** `lib/notify.js` (new), `app/api/intake/route.js`, `package.json`

The API route was persisting submissions to Supabase but never sending any alerts. Added a new notification module at `lib/notify.js` with:

- `sendEmailAlert(record)` — sends a formatted HTML email via nodemailer SMTP
- `sendTelegramAlert(record)` — sends a concise triage message via Telegram Bot API

Both functions are fire-and-forget: notification failure is logged to console but never throws, never blocks persistence, and never affects the response returned to the visitor.

The API route now runs:
```
persist → fire-and-forget notifications → return 200
```

`nodemailer@^6.9.16` and `@types/nodemailer@^6.4.17` were added to `package.json`. Run `npm install` locally or let Vercel install from the manifest.

---

### 4. OG image placeholder created
**Files:** `public/opengraph-image.svg`, `lib/site.js`, `app/layout.tsx`

A production-ready SVG Open Graph image was created at `public/opengraph-image.svg`. It carries the brand palette (dark navy + gold), headline copy, brand mark, and the non-law-firm disclaimer.

`site.js` was updated from `.png` to `.svg`, and the `openGraph.images` and `twitter.images` fields were added to the layout metadata so the image actually gets picked up by social scrapers.

> **Production note:** For best Twitter/OG compatibility, convert `opengraph-image.svg` to a 1200×630 PNG before launch. Many scrapers handle SVG, but PNG is safer. You can use Figma, Canva, or any image converter.

## What still needs manual setup before launch

### Required env vars (add to Vercel environment variables)

| Var | Purpose | Status |
|---|---|---|
| `SUPABASE_URL` | Supabase project URL | Required — must be set |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key | Required — must be set |
| `SUPABASE_INTAKE_TABLE` | Table name (default: `intake_submissions`) | Optional — default is fine |
| `LEAD_ALERT_EMAIL_TO` | Email address to receive lead alerts | Required for email alerts |
| `LEAD_SENDER_NAME` | Sender name in email headers | Optional — defaults to "Ontario Accident Review" |
| `SMTP_HOST` | SMTP server hostname | Required for email alerts |
| `SMTP_PORT` | SMTP port (default: 465) | Optional |
| `SMTP_SECURE` | `true` for TLS, `false` for STARTTLS | Optional — default `true` |
| `SMTP_USER` | SMTP auth username | Required for email alerts |
| `SMTP_PASS` | SMTP auth password or app password | Required for email alerts |
| `SMTP_FROM_EMAIL` | Sender email address | Optional — falls back to `SMTP_USER` |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token | Required for Telegram alerts |
| `TELEGRAM_CHAT_ID` | Telegram chat or channel ID | Required for Telegram alerts |

If `SMTP_HOST` or `LEAD_ALERT_EMAIL_TO` are not set, email alerts are silently skipped. Same for Telegram — if either var is missing, Telegram alerts are skipped. Submission persistence to Supabase is not affected either way.

### Supabase setup
Run the schema from `supabase/schema.sql` in your Supabase project SQL editor to create the `intake_submissions` table and its indexes. This is a one-time setup.

### Contact page placeholders

### Privacy and disclaimer pages

### OG image (PNG)
Convert `public/opengraph-image.svg` to a 1200×630 PNG for better social card compatibility. Update `site.js` `ogImage` to `/opengraph-image.png` after converting.

### Domain
